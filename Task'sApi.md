Задание: Создание сервера API для управления приложением 
Цель: Разработать server API на Express.js для управления приложение. API должно позволять
принимать get/post запросы для приложения wpf c#
Стек:
Backend: Node.js, next.js prisma orm, 
База данных: Postgress
Функциональные требования (Endpoints):
Ваш API должен поддерживать следующие HTTP методы и маршруты:
Создание задачи:
POST /tasks
Запрос: JSON объект с полями title (строка, обязательно) и description (строка, необязательно).
Ответ: Объект созданной задачи (включая уникальный ID) с статусом 201 Created.
Получение всех задач:
GET /tasks
Запрос: Нет.
Ответ: Массив всех задач с статусом 200 OK. Каждая задача должна включать поля id, title,
description, completed (булево, по умолчанию false), createdAt (дата создания).
Получение задачи по ID:
GET /tasks/:id
Запрос: Параметр id в URL.
Ответ: Объект задачи с указанным ID с статусом 200 OK. Если задача не найдена, 404 Not Found.
Обновление задачи по ID:
PUT /tasks/:id
Запрос: Параметр id в URL, JSON объект с полями title, description, completed (хотя бы одно из них).
Ответ: Обновленный объект задачи с статусом 200 OK. Если задача не найдена, 404 Not Found.
Частичное обновление задачи по ID:
PATCH /tasks/:id
Запрос: Параметр id в URL, JSON объект с любыми полями title, description, completed (одно или
несколько).
Ответ: Обновленный объект задачи с статусом 200 OK. Если задача не найдена, 404 Not Found.
Удаление задачи по ID:
DELETE /tasks/:id
Запрос: Параметр id в URL.

Ответ: Пустой ответ с статусом 204 No Content (если успешно). Если задача не найдена, 404 Not
Found.
